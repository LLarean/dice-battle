# План — Часть 1 (MVP)

> Цель: довести текущую игру до состояния «easy to play, hard to master».
> Кор остаётся простым. Усиливаем три вещи: **комбинации**, **читаемую обратную связь** (анимация суммирования), **уют антуража** + один лёгкий **мета-крючок**.
> Формат игры — «тыкалка в очереди»: партия на 1–5 минут, можно закрыть и продолжить.

## Принципы
- Не усложнять кор. Бросок → перебросы → результат — оставить как есть.
- Любую новую механику добавлять так, чтобы новичок понял её за один бой без чтения.
- Каждое действие должно давать петлю эндорфина: бросок → видимый эффект → исход → награда → повтор.
- Рандом — это фича, а не баг. Не строим тактику, которая требует точного контроля; строим «приятный риск».

---

## 1. Комбинации граней (главная механика глубины)

Сейчас `DiceResult.Calculate` просто суммирует грани (Attack/Defense/Heal) с множителями.
Добавляем правила за наборы одинаковых граней — это и есть «hard to master» при простом коре.

Предлагаемый набор (значения — на балансировку):
- **5 мечей (Attack)** → гарантированное убийство любого противника («казнь»).
- **5 сердец (Heal)** → полное восстановление здоровья.
- **5 щитов (Defense)** → полная броня / неуязвимость на ход.
- **4 одинаковых** → бонус-множитель к этому типу (например ×2 к сумме граней этого типа).
- **3 одинаковых** → небольшой флэт-бонус (например +2 к сумме этого типа).

Почему это ложится легко:
- Это проверка `Count` граней по `DiceValue` внутри уже существующего `Calculate`.
- Не требует новых типов кубиков и не ломает текущий инвентарь.
- Игрок сам на третьем-четвёртом бою замечает «о, если оставить мечи…» — это и есть скрытая глубина.

Связь с перебросами:
- Перебросы (уже есть) превращаются в осмысленный выбор: «добивать комбо или взять стабильную сумму».
- Это и есть риск/награда без дополнительных систем.

Открытые вопросы для баланса:
- 5 одинаковых из 5 кубиков — редкое событие; возможно «5 из имеющихся N» или «5 одинаковых подряд переброса». Решить на плейтесте.
- Гарантированное убийство против финального босса — оставить или ослабить до «×3 урон».

## 2. Анимация суммирования эффектов (читаемость + сок)

Сейчас результат применяется мгновенно. Добавляем простую, быструю визуализацию.

- После окончания хода: одинаковые грани **визуально слетаются и складываются** в одно число
  (три меча → «3» летит во врага в полоску здоровья/брони; сердца → в своё здоровье).
- Срабатывание комбо — отдельный заметный акцент (вспышка/иконка «КАЗНЬ», «ПОЛНОЕ ЛЕЧЕНИЕ»).
- Длительность — десятые доли секунды, чтобы не мешать «тыкалке».

Зачем:
- Делает математику боя **понятной без текста** — игрок видит, откуда взялось число.
- Даёт второй слой эндорфина (бросок + красивое срабатывание).
- Технически — слой поверх уже готового `DiceResult`, кор не меняется.

Реализация (ориентир, без обязательств): LeanTween (уже в проекте) для полётов чисел/иконок.

## 3. Сохранение боя (для формата «очередь»)

В `GameConfig` уже есть флаг `CanSaveBattle`, но сохранение боя не реализовано —
сейчас в `PlayerPrefs` живёт только прогресс по уровням (`CompletedLevels`/`CurrentLevel`) и инвентарь.

Нужно:
- Сохранять состояние **текущего боя**: здоровье игрока/врага, какой враг, оставшиеся перебросы,
  текущие грани кубиков, надетая дека.
- При закрытии/сворачивании приложения — автосейв.
- На стартовом экране кнопка **«Играть»** при наличии сейва ведёт **прямо в незаконченный бой**, иначе — обычный вход.
- После завершения боя сейв боя очищается (остаётся только прогресс).

Зачем: это прямое ядро сценария «играю в очереди, очередь подошла — закрыл, потом продолжил».

---

## 4. Антураж и уют (визуальная составляющая)

Без 3D и без новых сцен — приведение в порядок того, что есть.

- **Таверна + трактирщик**: довести как «домашний экран» между боями
  (приветствие трактирщика уже есть — `Innkeeper.ShowMessage`). Сделать тёплый, читаемый, без визуального мусора.
- **Согласовать надёрганные ассеты**: единая палитра/рамки/шрифты у города, таверны, боя.
  Это самая дешёвая по усилиям, но заметная по эффекту правка.
- **Фон боя под монстра** — уже есть (`UnitData.Background`), убедиться что выглядит цельно.
- **Простые эффекты**: удар, лечение, срабатывание комбо — лёгкие частицы/вспышки.

---

## 5. Мета-крючок (повод вернуться) — минимальный

Выбран лёгкий вариант, без переусложнения. Один из (решить на старте этапа):
- **Выбор персонажа в таверне** — за столами сидят классы (рыцарь/лучник/…),
  выбор даёт стартовую деку/небольшой пассив. Подбираем под настроение, не под бой.
- **ИЛИ лёгкая прокачка между забегами** — после победы/поражения немного «остаётся» (валюта/очко),
  тратится в таверне на расширение пула кубиков.

Важно: только **одна** мета-система в MVP, чтобы не размазать фокус.

Из исходной идеи:

- **3 любых → 1 случайный**.

Ставим ниже комбо и анимации: даёт смысл «лишним» наградам, но не критичен для первого интересного забега.

## Чек-лист готовности MVP
- [ ] Комбинации граней (3/4/5) работают и читаются игроком без объяснений.
- [ ] Анимация суммирования + акцент на комбо.
- [ ] Сохранение/продолжение боя из стартового экрана.
- [ ] Согласованный антураж таверны и боя, простые эффекты.
- [ ] Один мета-крючок (персонаж ИЛИ прокачка).
- [ ] (опц.) Рынок обмена кубиков.

## Сознательно НЕ в MVP
- 3D-сцены, проходы по городу/таверне.
- Глубокая тактика «дека под конкретного оппонента» (конфликтует с рандомом).
- Карта приключений, классовые способности, эффектные 3D-кубики — всё это во второй части.
